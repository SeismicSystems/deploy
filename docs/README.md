# Seismic Network Documentation

This directory contains comprehensive documentation for integrating with and operating on the Seismic Network.

## Documentation Files

### [`node-operator-faq.mdx`](./node-operator-faq.mdx)
**For: Node operators, developers, RPC providers**

Comprehensive technical FAQ covering:
- General network information and architecture
- Node setup, hardware requirements, and deployment
- RPC and API specifications
- EVM compatibility and custom features (shielded computation, SRC20 tokens, Sanvil)
- Gas model and transaction types
- Operations, maintenance, and monitoring
- Performance benchmarks and scalability

### [`custody-integration.mdx`](./custody-integration.mdx)
**For: Institutional investors, custodians, compliance teams**

Integration guide for institutional partners covering:
- Technical custody requirements and EVM compatibility
- Transaction construction and signing
- Token standards (ERC20 vs SRC20)
- Network participation (staking, governance - coming soon)
- Tokenomics and supply information
- Regulatory and compliance considerations
- Institutional support contacts

## Usage

These MDX files are designed for use with documentation tools like [Vocs](https://vocs.dev/). They include:
- Structured sections with anchored headings
- Code blocks with syntax highlighting
- Tables for quick reference
- Cross-references between related topics

## Coverage

The documentation answers questions from integration questionnaires for:
- Alchemy
- QuickNode
- Anchorage (custody)
- General blockchain integration partners

All technical questions about running nodes, integrating with RPC endpoints, and institutional custody have been addressed.
